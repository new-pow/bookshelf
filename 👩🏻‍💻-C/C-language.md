# C언어의 개요
## C언어의 등장과 발전
- 프로그래밍이란?
	- 컴퓨터가 작업을 수행하기 위해 실행해야 하는 일련의 명령을 작성하는 것
- 프로그래밍 언어
	- 프로그램을 작성하기 위해 사용되는 언어
	- 층위
		- 제1세대 언어(기계어): 2진수
		- 제2세대 언어(어셈블리어): 2진수 언어를 사람이 이해할 수 있는 언어로 표현한 것. 어셈블러를 통해 기계어로 변환하게 된다.
		- 제3세대 언어(고급언어): C, C++, Java 등. 사람이 작성하기 편한 언어. 컴파일러, 인터프리터로 어셈블리어로 변환한다.
- C언어의 연혁
	- 1960 ALGOL 60
	- 1963 CPL
	- 1967 BCPL
	- 1970 B언어
	- 1972 C언어
	- 1978 K&R C
	- 1989 ANSI C(C89)
	- 1999 C99
	- 2011 C11
	- 2017 C17
	- 2023 C23
	- ...
## C언어의 특징
- 범용 고급언어
	- 논리적인 처리 구조를 만들 수 있는 데이터 표현 및 흐름제어를 할 숭 ㅣㅆ음
	- 여러가지 유용한 처리를 위한 표준 라이브러리를 제공함
	- 프로그램의 이식성이 높음
- 시스템 프로그래밍에 적합한 언어
	- 저급언어 수준의 특성이 있으며, 하드웨어 제어에 적합함
	- 매우 빠른 실행 속도
- 간결한 문법 구조로 되어 있음
## C 프로그램의 작성
- 프로그램 개발 단계
	- 프로그램의 목적 정의
	- 프로그램 설계
	- 소스코드 작성
	- 컴파일/링크
	- 프로그램 실행
	- 테스트와 디버깅
	- 유지 보수
- C프로그램을 실행 파일로 만드는 과정
	- 코딩: 소스파일로 코딩한다. `.c`
	- 컴파일: 소스파일로 목적파일로 변경 (기계어로 변경됨) `.obj`
	- 링크: 라이브러리, 컴파일에서 미리 제공된 소스코드를 연결시켜 실행파일로 만듬. `.exe`
	- 빌드 : 컴파일 + 링크 단계를 합쳐서 이르는 말.
- 필요한 도구
	- 통합개발환경 IDE
		- 소스 코드 편집기
		- 컴파일러
		- 링커
		- 디버깅 도구
- C 프로그램 작성시 일반적인 규칙
	- 반드시 하나 이상의 함수를 포함해야함.
	- main() 함수가 반드시 존재해야한다.
	- 함수의 몸체는 시작과 끝을 알리는 중괄호 {} 를 사용하여 블록으로 구성한다.
	- 블록 안에는 변수 선언문, 치환-연산-함수호출 등의 명령을 기입한다.
	- 선행처리 지시어 를 제외한 문장의 끝에는 세미콜론 ; 을 붙인다.
- C 프로그램을 구성하는 토큰의 종류
	- 예약어: C에서 고유한 문법 및 의미가 정해진 단어
	- 명칭: 변수, 함수 등을 식별하기 위하 정의하는 이름
	- 상수: 값이 변하지 않는 자료(정수, 실수, 문자 등)
	- 문자열: 큰 따옴표로 묶인 문자 시퀀스
	- 구두점: 고유한 문법 및 의미가 정해진 기호
	- 설명문: 프로그램에 대한 주석
- 명칭 identifier
	- 정의 규칙
		- 영문자(대-소문자 구분함)와 숫자의 조합이어야함
		- 명칭의 첫문자는 영문자나 밑줄 _ 이어야 한다
		- 특수문자는 사용할 수 없다. 밑불은 사용가능
		- 문자사이에 공백이 있을수 없다.
- 상수 constant
	- 수치상수: `123`
	- 문자상수: ``a``
	- 문자열 상수: `"KNOW"`
- 연산자 operator
	- 여러가지 유형의 연산을 표현
	- 연산의 대상을 피연산자로 함.
- error
	- 프로그램을 정상적으로 빌드할 수 없는 문제가 포함된 소스코드의 경우 에러메시지를 출력함.
	- 에러 발생시 빌드가 중지되어 목적 코드 및 실행 파일이 생성되지 않음
- 경고 warning
	- 빌드를 진행할 수는 있지만, 문제 발생 가능성이 있는 소스 코드를 작성한 경우
	- 경고가 있더라도 실행 파일은 생성되며, 프로그램을 실행할 수 있음.
	- 경고 메시지의 내용을 파악하여 필요한 경우에는 적절히 수정하며, 아무런 문제가 없는 경고는 무시할 수 있음.

# 자료형
## 자료형
- 자료형 data type
	- 컴퓨터에서 값의 표현 방법을 정의한 것
		- 값의 종류에 따른 표현 방법
		- 값을 표현하기 위한 메모리 공간
		- 값을 대상으로 수행할 수 있는 연산
	- 값을 저장하거나 계산할 때 자료형을 엄격하게 구분하여 처리함
		- 필요한 경우 자료형을 변환하여 올바른 처리가 이루어질 수 있게 해야함.
### 자료형의 종류
- 기본형
	- 정수형
		- `short int`
		- `int`
		- `long int`
		- `long long int`
		- 양의 값만 사용할것인지, 아닌지에 따라 `signed`, `unsighed` 로 나뉨.
		- default 는 signed
	- 문자형
		- `char`
		- ASCII 코드인지, 아닌지(기타 문자들)에 따라 `signed`, `unsighed` 로 나뉨.
	- 실수형
		- `float`
		- `double`
		- `long double`
- 열거형
	- enumerated type
	- 이름을 나열하여 그를 값처럼 사용함
- 파생형
	- 배열형 array
	- 구조체형 structure
	- 공용체형 union
	- 포인터형 pointer
### 기본 자료형의 크기와 값의 범위
- 값을 저장하기 위한 메모리 크기
	- char <= short <= int <= long <= long long
- 메모리 크기가 확정되어 있지는 않으며, C언어를 구현하는 컴퓨터에서 가장 효율적으로 처리할 수 있는 정수형을 int 형으로 표현함.

## 상수
- 항상 고정된 값을 갖는 자료
- 자료형에 따라 정해진 문법에 따라 표현함.
- 정수형 상수
	- 일반적으로 10진 상수
		- 0~9범위의 숫자. 숫자 첫자리는 0이 아니어야 함.
	- 8진 상수 012, -055
		- 0~7범위의 숫자.
		- 숫자 첫자리는 0으로 시작.
	- 16진 상수 0x0c, -0X2D
		- 0~9, a~f 를 사용함.
		- 숫자 첫자리는 0x 로 시작함
	- unsigned형
		- 10u, 012u, 0X0CU
			- 뒤에 u를 붙임.
	- long 형
		- 12345l, 0FFL
			- 뒤에 l을 붙임.
	- 소수형식
		- 소수점을 사용
	- 지수형식 10진수와 e를 사용하여 표현
		- 1.2e3 = 1.2 * 10^3
	- float 형
		- 뒤에 f를 붙임
	- long double
		- 뒤에 l을 붙임
	- null 문자 '\0'
- 문자열 상수
	- 큰따옴표로 묶여있는 일련의 문자
	- 문자열의 끝을 나타내기 위해 null 문자가 추가됨. (`\0`)
## 변수
- 프로그램에서 값을 저장하기 위한 공간
- 모든 변수는 사용하기 전에 미리 선언되어야 함.
- 특징
	- 이름이 있음.
	- 정해진 자료형이 있음.
	- 할당된 값을 가짐.
```
int a;
double b;
```
- 정의를 하면 빈 메모리가 할당됨.
- 값을 저장하게 되면 해당 메모리가 채워짐
```
a = 3+2;
```
- 변수 선언시 고려할 사항
	- 변수에 저장될 값의 크기
	- 변수의 선언 위치
	- 변수의 초기화
### 저장될 값의 크기에 따라 발생할 수 있는 문제
- 오버플로 overflow
- 언더플로 underflow
- 표현할 수 있는 값의 범위를 벗어남.
### 변수 선언 위치
- 변수를 선언하는 위치에 따라 변수의 의미가 달라짐
- 전역변수
	- 프로그램의 어느 부분에서든 사용할 수 있음.
- 지역변수
	- 함수 안에서 정의된 변수의 경우, 해당 중괄호 안에서만 사용할 수 있음.
### 변수의 초기화
- 변수의 값을 사용하기 전에 적절한 값이 저장되어 있어야함.
- 없으면 안됨
# 선행처리기
- preprocessing
- 컴파일에 앞서 소스 프로그램을 가공하여 컴파일러가 실제로 번역할 소스 프로그램을 만드는 것
```c
#include <studio.h> // 선행처리

int main() {...}
```
- 선행처리 지시어 `#`
	- 한 행에 한 개의 문자를 작성함.
	- 문장 끝에 `;` 는 사용하지 않음
```c
// 헤더 파일 포함
#include <파일명>
#include "파일명"

// 매크로 선언 및 해제
#define 매크로명 값
#define PI 3.14 // 정의
#define 매크로명(인수리스트) (수식)
#undef 매크로명 // 해제

// 조건부 컴파일
#if
#else
#elif
#endif
```
- 매크로란?
	- 특정 코드 패턴으로 치환되도록 정의된 명칭
	- 자주 사용되는 명령이나 수식 또는 상수에 이를 대표하는 이름을 붙여서 사용할 수 있음.
- 매크로 확장
	- 인수를 수식에 반영하여 매크로명을 치환함
- 매크로 주의
	- 매크로명이 수식의 텍스트로 치환되는 것이므로 괄호를 적절히 사용할 필요가 있음
```c
#define C_AREA(x) (3.13 * (x) * (x))
// 사용하는 곳에서 x가 그대로 동일하게 들어감
```
