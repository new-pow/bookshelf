# 1. 컴파일러란?
## 컴파일러의 개념
- 컴파일러란?
	- 언어를 번역한다. 즉, 번역기
- 번역기의 종류
![](https://i.imgur.com/zbxAG5B.png)
- 어셈블러: 어셈블리어 -> 기계어
- 컴파일러: 고급언어 -> 저급언어(어셈블러)
- 인터프리터: 고급언어 -> 실행결과
- 프리프로세서: 고급언어 -> 고급언어
	- 언어를 업그레이드(버전업)할때 사용
## 컴파일러와 인터프리터
- **컴파일러 기법**
	- 번역 후 실행
	- 특징
		- 효율적
		- 반복문 처리에 효과적
		- 큰 기억장소 요구
	- 예시언어
		- C, C++, JAVA, COBOL 등
- **인터프리터 기법**
	- 번역과 실행
	- 특징
		- 실행시간이 길다
		- 사용자와 대화식
		- 융통성 있음
	- 예제언어
		- SNOBOL, LISP, APL 등
## 컴파일러의 논리적 구조
### 논리적 구조
1. 어휘분석
	- 기본어휘가 문법에 맞는지 분석
	- 어휘를 토큰으로 변환
	- 연산자, 식별자, 예약어, 구분자, 상수
	- 단어 분석: 변수 사용 등.
2. 구문분석
	- 문장 분석: 올바른 문법인지 분석
3. 의미분석
	- 기억장소
4. *중간코드*
5. *최적화*
6. 목적코드 생성
