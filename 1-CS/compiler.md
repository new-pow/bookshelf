# 1. 컴파일러란?
## 컴파일러의 개념
- 컴파일러란?
	- 언어를 번역한다. 즉, 번역기
- 번역기의 종류
![](https://i.imgur.com/zbxAG5B.png)
- 어셈블러: 어셈블리어 -> 기계어
- 컴파일러: 고급언어 -> 저급언어(어셈블러)
- 인터프리터: 고급언어 -> 실행결과
- 프리프로세서: 고급언어 -> 고급언어
	- 언어를 업그레이드(버전업)할때 사용
## 컴파일러와 인터프리터
- **컴파일러 기법**
	- 번역 후 실행
	- 특징
		- 효율적
		- 반복문 처리에 효과적
		- 큰 기억장소 요구
	- 예시언어
		- C, C++, JAVA, COBOL 등
- **인터프리터 기법**
	- 번역과 실행
	- 특징
		- 실행시간이 길다
		- 사용자와 대화식
		- 융통성 있음
	- 예제언어
		- SNOBOL, LISP, APL 등
## 컴파일러의 단계
1. **프로그램**
	- 고급 언어가 잘못되지 않았는지 확인.
	- 변수 시그니처 규칙
2. 어휘분석
3. 구문분석
4. 의미분석
5. 목적코드 생성
